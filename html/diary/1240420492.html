Wed Apr 22 2009 - New NewScript Compiler

- Dave Goehrig

I've added a number of major changes today to the beta code. Most significantly, the environment is now properly compiling multiple objects in the dictionary to an intel native binary object format. For the time being the code is limited to 1 page (4096 bytes) of executeable code, but that is sufficient for testing purposes. I'm personally less interested in producing applications at this point as proving out the code.

Today's changes include:

  • New registration code w/ default icon selection bar
  • Improved color-higlighting of objects and verbs
  • Firth compiler now compiles the full lexicon

There are a few known bugs that haven't been fixed yet:

  • It is possible to define an object or verb with a space in its name, which can not be executed due to how the NewScript object parser parses strings
  • Some people are still registering without user icons, making their channel icons invisible
  • A few opcodes are generating incorrect x86 assembler OR are not evaluating the same way in the emulator

Some changes that are in development that haven't been pushed live:

  • Javascript color highlighting code
  • NewScript -> Javascript bridge code, call Javascript from NewScript
  • NewScript emulator object redefinition support

The main reason these changes haven't been pushed live was that I've been running into errors where builtin Javascript objects have properties which can not be inspected without triggering permissions errors. This isparticularly true of HTTPRequest objects, which blow up with wonderfully cryptic error messages in different browsers.
